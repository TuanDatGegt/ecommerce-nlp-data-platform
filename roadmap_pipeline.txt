7. daily_monitoring
Features:

- Task dependency graph
- Logs each stage
- Manual rerun
- Email alert (optional)
================================================================================
STAGE 1: BRONZE LAYER - RAW INGESTION
================================================================================
Input:
37 TSV files
Process:
1. Read files in chunks
2. Standardize schema
3. Convert TSV -> Parquet
4. Compress storage
5. Add metadata:
   - source_file
   - ingest_time
Tool:
- Python
- DuckDB
- Pandas (optional)
Output:
/bronze/reviews/year=YYYY/month=MM/*.parquet
Storage:
- Local MinIO
or
- AWS S3
Why:
- Faster query than TSV
- Better storage efficiency
================================================================================
STAGE 2: SILVER LAYER - CLEANING & VALIDATION
================================================================================
Tool:
DuckDB querying parquet directly
Validation Rules:
1. Remove null product_id
2. Remove null review_text
3. Remove invalid ratings (<1 or >5)
4. Remove duplicate review_id
5. Remove rows:
   helpful_votes > total_votes
Text Cleaning:
1. Strip HTML tags
2. Lowercase normalize
3. Remove strange symbols
4. Detect language
5. Keep English only
Error Records:
Invalid rows sent to:
/dlq/error_rows.json
Output:
/silver/reviews_clean/*.parquet
================================================================================
STAGE 3: FEATURE ENGINEERING
================================================================================
Structured Features:
- review_length
- helpful_ratio
- rating_bucket
- sentiment_proxy
- review_age_days
Text Features:
- cleaned_text
- tokens
- ngrams (optional)
Output:
/gold/features/*.parquet

================================================================================
                         MEDALLION ARCHITECTURE
================================================================================

BRONZE LAYER
------------
Purpose:
Store immutable raw review data.

Flow:

Kaggle API
    ↓
ZIP Dataset Download
    ↓
Upload ZIP to MinIO RAW Bucket
    ↓
Extract TSV Files
    ↓
Chunk Processing
    ↓
Convert TSV → Parquet
    ↓
Upload to MinIO Bronze Layer

Storage Layout:

raw/
└── amazon_reviews/
    └── archive.zip

bronze/
└── reviews/
    └── year=YYYY/
        └── month=MM/
            └── category=Books/
                ├── part_0000.parquet
                └── part_0001.parquet


================================================================================
                          SILVER LAYER
================================================================================

Purpose:
Clean and validated review data.

Validation Rules:

1. Remove null product_id
2. Remove null review_text
3. Remove invalid ratings
4. Remove duplicate review_id
5. Remove corrupted rows
6. Remove:
   helpful_votes > total_votes

Text Cleaning:

1. Strip HTML
2. Lowercase normalize
3. Remove symbols
4. Detect language
5. Keep English only

Output:

silver/
└── reviews_clean/


================================================================================
                          GOLD LAYER
================================================================================

Purpose:
Analytics-ready business features.

Features:

- review_length
- helpful_ratio
- sentiment_proxy
- rating_bucket
- review_age_days

Output:

gold/
├── features/
├── product_metrics/
└── customer_metrics/

        ↓

        ↓

        ↓
vector_build_dag

================================================================================
                     BRONZE PIPELINE FLOW
================================================================================

STEP 1:
Authenticate Kaggle API

STEP 2:
Download Dataset

STEP 3:
Upload Raw ZIP to MinIO

STEP 4:
Extract TSV Files

STEP 5:
Read TSV in Chunks

STEP 6:
Optimize Data Types

STEP 7:
Add Metadata:
- source_file
- ingest_time

STEP 8:
Validate Schema

STEP 9:
Convert to Parquet

STEP 10:
Upload Bronze Parquet to MinIO

STEP 11:
Track Checkpoint

STEP 12:
Cleanup Temporary Files

================================================================================

      ↓
FastAPI /chat
      ↓
Embedding Model
      ↓
Convert Query → Vector
      ↓
Search ChromaDB
      ↓
Retrieve Top-K Reviews
      ↓
Build Prompt Context
      ↓
Send to LLM
      ↓
Generate Grounded Answer
      ↓
Return Response
